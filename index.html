Student Performance Analysis

This project explores how study time, attendance, sleep, motivation, parental involvement, and access to resources affect student exam scores.

Report

Main Findings

The data shows that the average exam score is around 67. Most students are grouped in the middle score ranges, which suggests that many students perform at an average level.

One interesting thing I noticed is that students who study more do not always perform better than those who study less. In some cases, students who skip classes seem to have more time to study, which may affect this pattern.

Other factors such as parental involvement, motivation level, and access to resources showed only small differences in performance. Sleep hours also did not show a strong pattern in this dataset.

Overall, attendance appears to be more consistent in relation to performance, while study time alone does not fully explain the differences in exam scores.

01 Cleaning the data with Python

I started by cleaning the dataset in Python. I checked the columns, handled missing values, and made sure the data was ready for analysis. This helped me understand the structure of the dataset before moving to the next step.

02 Uploading the data to Supabase

After cleaning the dataset, I uploaded it to Supabase so I could store it in a PostgreSQL database. This made it easier to query the data using SQL and gave me a more realistic analytics workflow.

query_stats

03 Querying and exploring with SQL

I used SQL to group data, calculate averages, compare categories, and explore relationships between variables. This step helped me move from raw data to actual insights that could be visualized.

bar_chart

04 Building the dashboard in Metabase

I connected Supabase to Metabase and built the dashboard using different charts. At first I had a connection issue, but I solved it by using the session pooler. Once that was fixed, I could create visuals and organize the dashboard properly.

monitoring

05 Enjoying the analytics process

I enjoyed this project because it felt like solving a problem step by step. I liked turning raw data into something meaningful and being able to explain what the numbers were saying through the dashboard.
